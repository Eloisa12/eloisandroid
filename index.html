Automation

    During my internship at Microsoft, I built three scripts to help automate the maintenance of Azure infrastructure.

    Before, when problems happened, on-call engineers had to manually change network settings or touch the nodes in the network. This process was tedious and not safe with many steps, it is possible to miss out one detail and cause severe incident.

    The scripts I made allow engineers to perform these actions with just few inputs and one click. The scripts also automatically send the changes to an internal review system, where a manager must approve them before they take effect. This made the process much faster and safer.

    I was able to read through and understand a large codebase and successfully integrated my scripts into the existing system. During and after development, I documented each step and potential pitfalls, and wrote five troubleshooting guides to support future maintenance and onboarding.

Sed

Source Code

    This script is written in Python. It emulates the functionality of the Unix sed command: find and change text in a file or text. Imagine a file with 1 million words, and you want to replace all word "cat" to word "dog" instead of openning up the file and manually change every words, sed can do it with one click. This project demonstrates my ability to build non-programmer friendly tools in python that streamline repetitive tasks and improve workflow efficiency.

    I also applied test-driven development practices to ensure the tools were reliable and easy to maintain. As a result, my tests cover 96% of the code, meaning that 96% of the code has been tested and works as expected.
